# Lessons — online-go-school

## 2026-06-13: E2E緑でも実ユーザーで死ぬ「状態乖離」パターン
**事象**: 先生PWサーバー再設定後、三村さんが正しい新PWでログイン不能。E2Eは13/13緑だった。

**根因**: ローカル localStorage のPW照合がサーバー認証より先に走るゲートが残っていた。
E2Eヘルパーは毎回 `setupTeacherPassword(TEST_TEACHER_PASSWORD)` でローカルとサーバーを
**同じ値に揃えて**いたため、「ローカル≠サーバー」という実ユーザーの状態を一度も踏まなかった。

**教訓**:
1. **二重に保存された真実（ローカルキャッシュ vs サーバー）があるなら、E2Eに必ず「乖離ケース」を入れる**。
   揃った状態だけのテストは、揃っていることのテストにしかならない。
2. サーバー側の認証情報を変更したら、「既存ブラウザ状態（古いキャッシュ）からの初回ログイン」を必ず実機 or E2Eで踏む。
3. 認可の権威を移行するときは、旧権威のコード（照合・リセットUI）を**残さず撤去**する。
   並存させると「どちらかが通れば入れる/どちらかが落ちれば弾く」の混在で必ず事故る。
   → 対応済み: `e2e/teacher-auth.spec.ts` に乖離ケース2本を常設（commit 481b962）。

## 2026-06-30: 競合分析でブランド資産を機能比較に矮小化しない
**事象**: online-go-school の市場比較で、横浜囲碁サロン・ダイヤモンド囲碁サロン・洪道場の強みを「商品設計」「講師陣」「本格派」程度に薄くまとめてしまった。

**根因**: 競合の選ばれる理由を、機能・価格・提供形態に寄せすぎた。実際には、横浜囲碁サロンは囲碁YouTubeでの成功と発信者への信頼、ダイヤモンド囲碁サロンは都心一等地の高級サロンと女性アマチュアインストラクターの魅力、洪道場は日本最強棋士を何名も排出した実績ブランドが中核。

**教訓**:
1. 教室・サロン・教育サービスの競合分析では、機能表の前に「なぜその人・場が選ばれているか」を書く。
2. ブランド、人物像、場所の空気、実績、継続接触は、価格や機能と同等以上の購買理由として扱う。
3. 三村側の比較では、システム利便性だけでなく、三村九段の人柄・教室の継続感・保護者の安心感を必ず評価軸に入れる。

## 2026-07-08 「スクショで確認済み」と言いながら画像を見ていなかった
- **事実**: PWAボタンの本番検証でinnerText検出とconsoleログだけを見て「両画面で確認済み」と報告。実際は①先生の実動線（ログイン直後の教室管理画面）にボタンが存在しない②下部バーが#rootのpadding分（64px）ビューポート外に見切れ、の2点を見逃した。三村さんの実機指摘で発覚。
- **教訓**: スクリーンショットを撮ったら必ず自分の目で画像を開いて見る。テキスト存在≠ユーザーに見えている。「ユーザーが実際に通る動線」（ログイン後にどの画面に着地するか）で確認する。見切れはgetBoundingClientRect vs innerHeightで機械検証できる。
