# Windows版パリティ監査

**目的**: 元Windowsソフトの見た目だけでなく、授業で必要な操作をブラウザ版で同等に動かす。

**進め方**:
1. Windows版の録画・スクショ・サンプルファイルから正解挙動を復元する。
2. ブラウザ版の各ボタンを「実装済み / 部分実装 / 見た目だけ / 不要候補 / 要Windows確認」に分類する。
3. 優先度の高いものから1機能ずつ修正する。
4. 1機能ごとにビルド、対象テスト、Playwrightスクリーンショット確認を行う。

## 状態定義

| 状態 | 意味 |
|---|---|
| 実装済み | ボタンや入力が実データ・画面遷移・保存までつながっている |
| 部分実装 | 基本動作はあるが、Windows版との差分や運用上の穴がある |
| 見た目だけ | UIはあるが `onClick` 等の実処理がない、または効果が確認できない |
| 不要候補 | Windows版にはあるがブラウザ版では削る可能性がある |
| 要Windows確認 | Windows版の正解挙動が未確認 |

## 初回に直す候補

1. `回線復旧`: 授業中に最も事故を救うボタン。LiveKit reconnect / audio start / participant refresh に接続する。
2. `ビデオリセット`: カメラ・映像トラックを再公開し直す。
3. `共有検討`: 生徒対象選択、盤上マーク、分岐、戻る/進む、描画クリアまで実レッスン単位で検証する。
4. `対局作成`: 生徒1名との対局作成から初手同期、先生代打ち初手まではE2E通過済み。複数生徒・終局処理を追加監査する。
5. `XMLインポート`: Windows版実XMLサンプルで完全対応する。
